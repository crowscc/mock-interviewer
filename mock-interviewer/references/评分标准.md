# 评分标准

## 一、四维度评分细则

### 技术深度（5分制）

| 分数 | 特征 |
|------|------|
| 5 | 不仅理解原理，还能讲清演进历史、对比方案优劣、预判技术趋势 |
| 4 | 理解原理，能结合实际项目说清楚为什么这样做、遇到什么坑 |
| 3 | 会用，能说清基本原理，但追问2层就开始模糊 |
| 2 | 停留在"用过"层面，追问具体原理就答不上来 |
| 1 | 简历上写了但实际不了解，或者明显背的八股文 |

### 表达结构（5分制）

| 分数 | 特征 |
|------|------|
| 5 | 结论先行，分层清晰，详略得当，能根据面试官反应调整表达 |
| 4 | 有结构感（先总后分），表达流畅，重点突出 |
| 3 | 基本能说清楚，但有时会发散或遗漏重点 |
| 2 | 想到哪说到哪，缺乏结构，需要面试官反复追问才能拼出完整信息 |
| 1 | 表达混乱，逻辑不通，或过度简短无法传递有效信息 |

### 业务sense（5分制）

| 分数 | 特征 |
|------|------|
| 5 | 能从业务目标反推技术方案，算清ROI，主动关联商业价值 |
| 4 | 理解业务背景，能说清技术决策和业务目标的关系 |
| 3 | 知道业务背景，但技术和业务的关联讲得不够紧密 |
| 2 | 只关注技术实现，被问到业务价值时才勉强关联 |
| 1 | 完全技术视角，不理解或不关心业务目标 |

### 应变能力（5分制）

| 分数 | 特征 |
|------|------|
| 5 | 追问/压力题下依然稳定，能拓展思考，甚至能反问面试官形成讨论 |
| 4 | 被追问时能合理延伸，不确定的地方坦诚说并给出思考方向 |
| 3 | 基本能应对追问，但偶尔会慌或给出模糊回答 |
| 2 | 被追问就开始紧张，容易自相矛盾或过度防御 |
| 1 | 追问即崩，无法承受压力，或反复说"不知道"没有任何尝试 |

## 二、各面试官的维度权重

| 面试官 | 技术深度 | 表达结构 | 业务sense | 应变能力 |
|--------|---------|---------|----------|---------|
| P7 技术面 | 40% | 20% | 15% | 25% |
| P8 技术面 | 30% | 25% | 20% | 25% |
| P9 技术面 | 25% | 25% | 25% | 25% |
| P8 业务面 | 15% | 25% | 40% | 20% |
| P9 业务面 | 10% | 25% | 40% | 25% |
| VP/CTO | 15% | 25% | 30% | 30% |

**加权总分计算：**
```
加权总分 = 技术深度×权重 + 表达结构×权重 + 业务sense×权重 + 应变能力×权重
```

## 三、Level 对标特征

### P7 水平候选人的回答特征

**技术深度：**
- 能说清自己用的技术的基本原理
- 知道常见坑和解决方案
- 但缺少横向对比，不太清楚替代方案

**表达结构：**
- 能把事情说清楚，但有时会过于纠结细节
- 结构化程度一般，需要面试官引导

**业务sense：**
- 知道业务背景，但不太主动关联业务价值
- 被问到ROI时需要临时计算

**应变能力：**
- 追问1-2层能应对
- 超出准备范围容易紧张

---

### P8 水平候选人的回答特征

**技术深度：**
- 能做技术选型对比，说清楚每个方案的优劣
- 不仅知道怎么做，还能说清为什么这样做
- 有踩坑和解决问题的具体案例

**表达结构：**
- 结论先行，有清晰的框架感
- 能控制答题节奏，重点突出
- 会主动补充关键上下文

**业务sense：**
- 能主动把技术决策和业务目标关联
- 知道核心指标，能算清投入产出
- 能说清为什么做这个不做那个

**应变能力：**
- 追问2-3层依然稳定
- 不确定的地方坦诚说，但会给出思考方向
- 能接住压力题，给出有逻辑的回答

---

### P9 水平候选人的回答特征

**技术深度：**
- 有技术判断力，能评估技术方向的长期价值
- 跨领域知识储备，能做跨系统的架构设计
- 对技术发展趋势有自己的判断

**表达结构：**
- 高度结构化，能根据受众调整表达方式
- 善用类比和可视化让复杂问题简单化
- 答题有节奏感，能主动引导话题走向

**业务sense：**
- 从业务目标反推技术方案
- 能做资源取舍的量化决策
- 理解竞争格局和行业趋势对技术方向的影响

**应变能力：**
- 任何追问都能接住，甚至能反问面试官形成高质量讨论
- 对不确定的问题能给出分析框架而非具体答案
- 能把压力题转化为展示深度思考的机会

## 四、常见扣分项

| 扣分项 | 扣分维度 | 说明 |
|--------|---------|------|
| 背诵感 | 技术深度 -1 | 明显是背的答案，追问就卡 |
| 答非所问 | 表达结构 -1 | 面试官问A，答的是B |
| 夸大数据 | 业务sense -2 | 数据经不起追问，前后矛盾 |
| 甩锅 | 应变能力 -1 | "这个不是我负责的"、"是其他团队的问题" |
| 过度防御 | 应变能力 -1 | 被质疑时情绪化或强行辩解 |
| 假装知道 | 技术深度 -2 | 不懂装懂，不如坦诚说不了解 |
| 没有数据 | 业务sense -1 | 所有成果描述都没有量化 |
| 全程被动 | 表达结构 -1 | 只回答不展开，需要面试官不断追问 |

## 五、加分项

| 加分项 | 加分维度 | 说明 |
|--------|---------|------|
| 主动延伸 | 表达结构 +1 | 回答完后主动补充相关思考 |
| 坦诚不足 | 应变能力 +1 | 说清楚自己不了解的部分，并给出学习方向 |
| 反问面试官 | 应变能力 +1 | 合理反问形成技术讨论 |
| 跨项目串联 | 业务sense +1 | 把不同项目的经验关联起来，展示系统性思维 |
| 失败案例 | 业务sense +1 | 主动分享失败经历和反思 |
