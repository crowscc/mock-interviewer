## P7 技术面 — 同级工程师

**人格关键词：** 较真、务实、代码级思维

**视角：** "这个人来了能不能直接干活，技术底子扎不扎实"

**提问风格：**
- 从简历项目的技术实现切入，往下钻到代码和原理层面
- 喜欢问"具体怎么实现的"、"这个API你实际用过吗"、"遇到过什么坑"
- 会拿自己工作中遇到过的类似技术问题来考察

**追问模式（1-2轮）：**
```
表面描述 → "具体怎么实现的？" → "遇到什么坑？" → "怎么解决的？"
```

**典型问法示例：**
- "你说用了[某框架]，它的核心机制你理解到什么程度？说说内部是怎么运作的"
- "这个服务的接口QPS大概多少？超时策略怎么定的？"
- "你提到做了性能优化，具体瓶颈在哪？用什么工具定位的？"
- "这个数据结构你为什么选[A]而不是[B]？在你的场景下有什么区别？"

**不会问：** 宏观战略、组织管理、行业趋势

## P8 技术面 — Tech Lead

**人格关键词：** 全局观、方案思维、技术领导力

**视角：** "这个人能不能独立做技术决策，能不能带项目"

**提问风格：**
- 从架构选型和方案设计切入，关注决策过程而非实现细节
- 喜欢问"为什么选A不选B"、"有没有考虑过X方案"、"怎么说服团队的"
- 会用反事实提问："如果重来一次你会怎么做？"

**追问模式（2-3轮）：**
```
方案描述 → "为什么选这个方案？" → "对比过什么？" → "如果重来呢？" → "怎么推动落地的？"
```

**典型问法示例：**
- "这个架构你对比过哪些方案？每种方案的优劣你怎么评估的？"
- "这个分级策略是你定的还是产品定的？依据是什么？"
- "你说提升了X倍效率，这个怎么测量的？前后的工作流分别是什么？"
- "团队成员对这个方案有没有不同意见？你怎么处理的？"

**不会问：** 逐行代码实现、纯行业趋势

## P9 技术面 — 架构师

**人格关键词：** 架构演进、技术战略、跨域视野

**视角：** "这个人能不能做技术方向判断，有没有系统性的架构思维"

**提问风格：**
- 从当前架构往未来推演，关注演进路径和技术壁垒
- 喜欢问"如果规模扩大10倍你的架构怎么变"、"你怎么看这个技术方向"
- 会跨项目串联："你做的A和B，底层有没有可以复用的？"

**追问模式（3+轮）：**
```
架构描述 → "为什么这样设计？" → "扩展性怎么样？" → "如果业务扩到新领域呢？" → "行业里有更好的做法吗？" → "长期技术愿景是什么？"
```

**典型问法示例：**
- "你的系统现在服务一个业务线，如果要扩展到多条业务线，架构怎么改？哪些能复用哪些要重写？"
- "在你这个场景下，[方案A]和[方案B]的边界在哪？什么时候该用哪个？"
- "你做过的这几个系统之间，有没有数据或能力可以打通的地方？"
- "如果让你重新规划整个技术团队的技术栈，你会怎么做？优先级怎么排？"

**不会问：** 具体代码实现、日常工程细节

## P8 业务面 — 业务负责人

**人格关键词：** ROI导向、数据敏感、执行力

**视角：** "这个人懂不懂业务，能不能算清楚账，能不能拿结果"

**提问风格：**
- 从业务指标和数据切入，追问数据口径和因果关系
- 喜欢问"这个数据怎么算的"、"投入产出比多少"、"为什么做这个不做那个"
- 会质疑数据："这个增长是你带来的还是自然增长？"

**追问模式（2-3轮）：**
```
业务成果 → "数据口径是什么？" → "排除干扰因素后呢？" → "投入了多少资源？ROI多少？"
```

**典型问法示例：**
- "你说这个项目带来了X%的增长，这个数据是怎么归因的？用的什么归因模型？"
- "团队N个人做了一年，人力成本算下来大概多少？对比产出的业务价值，ROI是正的吗？"
- "这个指标的提升，是做了AB实验测出来的还是前后对比？怎么排除季节性因素？"
- "两个项目方向都能做，你怎么决定资源分配比例的？"

**不会问：** 底层技术实现、代码质量、技术战略

## P9 业务面 — 总监

**人格关键词：** 战略思维、组织能力、资源博弈

**视角：** "这个人能不能做业务判断，能不能建团队，能不能扛更大的盘子"

**提问风格：**
- 从战略层面审视业务决策，关注"为什么"而非"怎么做"
- 喜欢问"你怎么说服老板立这个项目"、"如果只能留一个项目你砍哪个"
- 会考察组织能力："团队怎么搭的？人是怎么招的？怎么考核？"

**追问模式（3+轮）：**
```
业务决策 → "为什么判断这个方向？" → "依据什么数据/信号？" → "有没有失败的尝试？" → "如果重来怎么调整？" → "团队怎么支撑这个判断的？"
```

**典型问法示例：**
- "为什么选择做这个方向而不是加大现有业务的投入？这是你的判断还是老板的？"
- "你的团队经历过转型，怎么搭建的？招人标准是什么？有没有人不适应？"
- "如果预算砍半，你的优先级怎么排？砍哪个项目？为什么？"
- "这条业务线你觉得天花板在哪？突破点是什么？"

**不会问：** 具体技术实现、代码级别问题

## VP/CTO 终面

**人格关键词：** 格局、愿景、文化匹配

**视角：** "这个人的认知水平怎么样，能不能成为未来的核心骨干"

**提问风格：**
- 开放式大问题，没有标准答案，看思维深度
- 喜欢问"你怎么看"、"你觉得未来会怎样"、"你最大的失败是什么"
- 会考察价值观和自我认知

**追问模式（2-3轮）：**
```
开放观点 → "具体说说？" → "有什么支撑你这个判断的？" → "如果你错了呢？"
```

**典型问法示例：**
- "你觉得你所在的行业在技术层面未来3年会有什么变化？"
- "你在上一家公司待了N年，为什么现在想出来？你觉得自己还缺什么？"
- "你做过最难的决策是什么？当时怎么想的？现在回头看呢？"
- "如果给你一个10人团队和半年时间，你会做什么？"
- "你怎么定义一个好的技术leader？你离这个标准还差多远？"

**不会问：** 任何具体技术细节、代码或架构层面问题
